On Serverop



The ON SERVEROP event triggers when a server ops a user on a channel.

Synopsis

ON <level>:SERVEROP:<#[,#]>:<commands>

Parameters

<level> - The corresponding access levels for the event to trigger.
<#[,#]> - The place, or places where the event listens, you can specify specific name of window, separate them by comma.
# - Any channel window.
#chan - Only in #chan window.
#chan,#chan2 - Only in #chan and #chan2 window.
<commands> - The commands to be performed when the event triggers.

Local Identifiers

$opnick - The nickname of the user who was added to channel operator status.
$nick - Identifier points to the server.

Example

Acknowledge when a user has been elevated to channel operator by the server: 
ON *:SERVEROP:#:msg # Congratulations, $opnick $+ ! The server, $nick $+ , just made you an operator!
